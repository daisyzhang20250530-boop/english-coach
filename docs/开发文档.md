# 职场英语教练 — 开发文档

> 配套文档：`docs/PRD_v2.md`（产品需求）。本文档面向开发迭代，记录架构、机制、构建与部署流程。
> 最后更新：2026-07-06

## 1. 项目概览

基于刻意练习方法论的职场英语训练引擎。单页 React 应用，构建为**单文件 HTML**（零依赖运行），AI 采用 DeepSeek API（用户自备密钥），进度存浏览器 localStorage，可选 GitHub Gist 云同步。

### 仓库结构

```
index.html          线上版（无密钥，GitHub Pages 入口，首次使用提示输入密钥）
app/                源代码
  src/App.jsx       全部应用逻辑（单文件组件，~1300 行）
  src/main.jsx      入口 + window.storage 适配层
  vite.config.js    构建配置（vite-plugin-singlefile 打单文件）
  package.json
docs/
  PRD_v2.md         产品需求文档
  开发文档.md        本文档
scripts/发布.ps1     一键构建+上传脚本（Windows PowerShell）
```

**不在仓库里的东西（含密钥，永远不要上传）**：`app/.env.local`（DeepSeek 密钥）、`github令牌.txt`、本地版 `职场英语教练.html`（密钥已烤入）。

## 2. 架构：规则引擎管决策，大模型管内容

这是全项目最重要的设计原则。AI 只做两件事：**照规格出题**、**照守则判分**。所有教学决策（难度、复习时机、弱点追踪）都是确定性代码：

| 机制 | 实现位置（App.jsx 内搜索） | 规则 |
|---|---|---|
| 难度升降档 | `applyResult` | 连对2升级，连错2降级，L1-L5 |
| 弱点档案 | `errorMap` | 8 类错误标签计数，驱动复仇题 |
| 复仇题 | `revengeIdx` | 每块第3题针对最近错误类型 |
| 结构课程表 | `STRUCTURES` / `pickStructure` | 解码线 11 种句法结构，逐个追踪掌握度 |
| 结构迁移 | `lastStruct` | 答错的结构换内容重考直到攻克 |
| 表达卡池 | `expressionBook` / `mergeCards` | 判分自动抽词组卡；组句题强制使用；主动用对≥2次=已掌握 |
| SRS 间隔复习 | `SRS_DAYS` / `srsAdvance` / `isDue` | 卡片/结构/错题统一按 1→3→7→16→35 天调度，答错归零 |
| 错题重考 | `dueWrongFor` / `retestId` | 到期错题占用复仇位，换内容考同一弱点 |
| 素材池 | `materials` / `extractMaterial` | 真实工作材料提取难句/表达/话题；解码题 50% 用素材（一半原句一半同结构改编） |
| 预取 | `prefetchQuestion` / `genQuestionCached` | 答题时后台生成下一题；升降级/复仇/迁移不符时自动废弃重生成 |
| 双模型 | `AI_MODEL_GEN` / `AI_MODEL_GRADE` | 后台预取用 reasoner（质量），现场生成/判分用 chat（速度） |

## 3. 题目质量的第一性原理（改提示词前必读）

一道题 = 测量仪器 + 训练刺激。九条推论（详见对话迭代史，此处为执行摘要）：

1. **单一变量**：难度只来自目标结构/目标技能；词汇控制在 B2、内容用中文喂给（组句/接龙题不让用户构思内容）
2. **答案无争议**：正确项绝对忠实原文——不添施动者、不发明事实/截止时间/新承诺
3. **干扰项有诊断价值**：每个对应该结构诱发的一种具体误读
4. **无捷径**：选项必须比原句简单（≤12词大白话）；干扰项须商业上可信，不能靠常识排除
5. **变化中重复**：迁移题必须换话题换主干动词
6. **真实迁移**：草稿必须像"这个学习者"会写的（禁止母语者口头语病如 "like,"）；参考答案按媒介惯例（slide 碎片句合法）
7. **语气=意图匹配**：hedging 是工具不是错误；只有语气与意图错位才算错（rewrite 题带 intent 字段）
8. **测量解耦**：复习卡升降档只看短语本身用得对不对（phraseOk 独立裁决），不被其他错误污染
9. **提示词管语言质量，代码管规则执行**：能用代码强制的（如标签过滤）不依赖模型自觉

## 4. 本地开发环境

1. 安装 Node.js LTS（nodejs.org）
2. `cd app && npm install`
3. 开发预览：`npm run dev`（http://127.0.0.1:5173）
4. 密钥：`app/.env.local` 写 `VITE_DEEPSEEK_API_KEY=sk-...`（没有此文件也能跑，运行时会提示粘贴密钥）

## 5. 构建与发布（两版机制）

- **本地版**：`npm run build`（.env.local 存在时密钥烤入）→ dist/index.html 复制为桌面的 `职场英语教练.html`。**此文件含密钥，禁止上传。**
- **线上版**：先把 `.env.local` 改名移开再 build（密钥不烤入，运行时输入）→ 上传为仓库根的 `index.html`。
- 一键脚本：`scripts/发布.ps1` 自动完成两版构建 + 密钥泄露校验 + 上传 GitHub + 开启 Pages。令牌放在脚本同级目录外的 `github令牌.txt`。

**铁律：每次上传前 grep 检查线上版不含 `sk-` 密钥。**

## 6. 云同步

GitHub Gist 私密文件 `wec-progress.json` 作云端存档。应用内"开启云同步"粘贴 gist 权限令牌；启动拉取（谁新用谁 + 空进度防覆盖保险），每次答题防抖 2.5s 推送。
