# english-coach 迭代方法论

这个项目由一位代码小白但产品直觉一流的 PM 与 Claude 结对开发。两个月里的每次返工几乎都违反了下面某条原则——它们不是理论，是事故报告的结晶。

## 先读什么

改动前必读 `开发文档.md`（架构、机制清单、九条题目质量原理、构建发布流程都在里面）。本技能不重复它的内容，只管"怎么思考和行动"。

## 协作方式

- 用户习惯**先分析对齐方案、再写代码**。收到需求先给分析和方案，重大设计决策用选择题（AskUserQuestion）让她拍板，然后不间断地做完。
- 她的反馈——包括说不清原因的"这里感觉不对"——**几乎总是指向真实设计缺陷**。诊断是你的事，不要求她给出诊断。
- 对她解释时少用术语、多用类比，但不降低内容密度；她能消化第一性原理级别的讨论。

## 四条铁律

### 1. 反馈先升维（她亲自立的宪法）

收到任何反馈，先问：**这是个例，还是一类问题的实例？** 是类，就机制化——建表、建账本、建通用检查；禁止针对单点做硬编码补丁。
事故案例：用户说"反对时总忘缓冲开场"，第一反应是在判分里硬编码检查缓冲——被她纠正为"过拟合"。正解是话步系统（MOVES 表：6 种意图 × 话步公式 + 逐步判分 + 话步账本自动发现缺口）。
本项目的所有资产（结构课程表、SRS、忠实性规则、话步系统）都是升维的产物。

### 2. 测量卫生：单一变量

任何一次测量只允许一个变量起作用。历史上被抓过五次污染，改动前对照自查：
- 生词污染结构测量 → 出题词汇锁 B2
- 内容构思污染语言测量 → 中文要点喂给用户，只考"怎么说"
- 其他错误污染短语掌握度 → phraseOk 独立裁决
- 句长污染流利度 → wpm 归一化 + 同等级内比较
- 读题时间污染产出计时 → 读题阶段不计时
另两条同族原则：**测量与训练分离**（考试模式不写训练状态）；**证据校准**（不同题型的一次对错含金量不同——三判全对=连对2题的证据量；2/3 是边界噪声记中性。新题型上线前先定"这题值多少证据"）。

### 3. 题目 = 测量仪器 + 训练刺激

九条推论在 `开发文档.md` §3，改出题/判分提示词前必须重读。最常犯的三条：正确项绝对忠实（不发明施动者/事实/截止时间/新承诺）；选项必须比原句简单（用户解码的是句子不是选项）；干扰项须商业可信且各对应一种具体误读。

### 4. 提示词管语言质量，代码管规则执行

便宜模型不守多约束指令是常态，不是意外。凡能用代码强制的（标签过滤、类型转换、格式校验），绝不依赖模型自觉。提示词只负责代码管不了的：语言质量、判断力、内容生成。

## 新功能三问

1. **值不值一个题位？** session 只有 6 题，每个名额都要值回票价。信息密度低的题型要么加密（如语气判断三合一），要么淘汰。
2. **新技能怎么入门？** 升到新等级/新题型不能裸考。用脚手架渐撤（scaffolding with fading）：初期给支持（预标标签/放宽时限），达标后撤除。恐慌区和学习区的区别不是难度，是**有没有可执行的下一步**。
3. **教的是操作程序还是知识点？** 心理表征来自可执行的程序（逐句打标签→公式组装），不是"知道错了什么"。反馈要落在原文上做手术（句子解剖/段落解剖/话步打勾），不要只写评语。

## 发布纪律（每次改动后自动执行，不等提醒）

1. 运行 `发布.ps1`（两版构建 + 密钥泄露校验 + 上传 GitHub + Pages 更新）
2. **铁律：线上版含 `sk-` 密钥则中止**——脚本会查，但你要看它的输出确认
3. 迭代日志追加到 `开发文档.md` §7（漏过一次被用户抓过）
4. 提醒用户**关掉重开页面**——旧标签页跑旧版本，是"改了没生效"类问题的头号原因
5. 永不上传：`app/.env.local`、`github令牌.txt`、`职场英语教练.html`（含密钥）

## 工程坑位速查

- deepseek-reasoner 的 max_tokens **包含思考过程**，要给 6000；出题=reasoner（预取隐藏延迟），现场生成/判分=chat（用户在等）
- 预取缓存 `_qKeyOf` 必须覆盖每个影响出题的新参数，否则错题重考/考试会误用缓存
- PowerShell 5.1：脚本要 UTF-8 **带 BOM**；无 `&&`；`$env:X=""` 等于删除变量
- React：提前 return 必须放在所有 hooks 之后
- 模型返回的数字字段可能是字符串，比较前 `Number()` 转换
- **回调别直接挂 onClick**：`onClick={fn}` 会把点击事件传给 `fn` 的第一个参数，`arg || fallback` 里事件对象恒为真——07-10 的判分 bug 让所有题不论对错记错了三天。写 `onClick={() => fn()}`；带可选参数的函数要校验参数形状
- 用户报"等级卡死/反复出同类题"，先看导出数据：session 的 correct 计数 + 错题本里 userAnswer===reference 的矛盾记录，是判分记账 bug 的指纹
- 本地文件可能被意外回退（云盘同步/系统还原）：动手前先 grep 本地版本是否包含最近迭代的特征函数，仓库源码即备份
- 单人产品没有对照组：功能有效性只能靠"第一性原理定先验 + 考试成绩单做剂量-反应"判断，别许诺做不到的归因
